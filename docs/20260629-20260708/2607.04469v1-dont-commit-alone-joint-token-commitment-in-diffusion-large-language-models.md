## 摘要
扩散大语言模型（dLLMs）在每次去噪步骤中通过从共享上下文中独立地解码每个选定位置来承诺多个令牌；当这些位置相互依赖时，由此产生的分解误差由条件总相关性捕获，而基于置信度的选择无法仅从边际分布中观察到这一点。我们提出CoCommit，一种标记门控协调传递，它短暂推迟了承诺：在常规束选择之后，一个学习到的标记宣布承诺集，并且骨干网络的最后n层被重新应用，以便标记位置进行协调——近似联合模式解码——然后执行贪婪argmax写入令牌。该方法重用现有权重，只需一次额外的前向传递，且无需辅助模型。在使用LoRA适配器和匹配贪婪推理的LLaDA2.1-mini上，联合承诺在所有六个评估基准上提高了准确性，其中在推理和精确答案任务上提升最大。

## Abstract
Diffusion large language models (dLLMs) commit multiple tokens per denoising step by decoding each selected position independently from the shared context; when those positions are dependent, the resulting factorization error is captured by conditional total correlation, which confidence-based selection cannot observe from marginals alone. We propose CoCommit, a marker-gated coordination pass that briefly defers commitment: after the usual bundle selection, a learned marker announces the commit set and the backbone's last-$n$ layers are re-applied so marked positions coordinate -- approximating joint-mode decoding -- before greedy argmax writes tokens. The method reuses existing weights with one extra partial forward pass and no auxiliary model. On LLaDA2.1-mini with LoRA adapters and matched greedy inference, joint commitment improves accuracy on all six benchmarks we evaluate, with the largest gains on reasoning and exact-answer tasks.

## 1. 论文的核心问题与整体含义（研究动机与背景）

扩散大语言模型（dLLMs）通过在每一步去噪时并行提交多个 token 来实现快速生成，但每个 token 是从共享上下文中独立解码的，忽略了位置之间的依赖关系。作者用**条件总相关性（Conditional Total Correlation, TC）** 来刻画这种因子化误差：当提交的 token 集合存在依赖时，联合后验被因式分解为边际分布的乘积，这一近似误差就是 TC。置信度选择（committing positions with highest marginal confidence）只能看到边际分布，而边际分布是联合模式的平均，无法捕捉依赖，因此无法避免不一致的元组。论文指出，**误差不能在提交前通过选择消除，必须在提交时通过协调来处理**。CoCommit 正是在这一时刻引入协调机制，将独立 argmax 替换为近似联合模式 argmax。

## 2. 论文提出的方法论

### 核心思想
在提交之前，通过一个**标记门控的协调传递（marker-gated coordination pass）**，让被选中的 token 位置在骨干网络最后 n 层中再次进行注意力交互，从而在写入 token 前协调它们所来自的联合模式。这样，后续的贪婪 argmax 能够取自同一个联合模式，而非混合模式下的独立最大值。

### 关键技术细节
- **Stage A（选择阶段）**：使用标准置信度规则（如阈值 τ_m2t）选出提交束 S，但不写入 token。这一步沿用常规的前向传播，只读取因式化的 logits。
- **Stage B（协调阶段）**：将学习到的标记向量 m 加到 S 位置的隐藏状态上（其他位置不加），然后**重新应用**骨干网络的最后 n 层。被标记的位置通过注意力机制互相影响，从而打破模式平均化，产生协调后的 logits。
- **最终写入**：从 Stage B 的 logits 对每个位置执行贪婪 argmax，得到同一联合模式下的 token 元组。

### 训练过程
训练采用**分裂前向（split forward）** 设计：
- **共享前缀**：先计算除最后 n 层以外的前缀隐藏状态 h。
- **Branch 1（选择分支）**：不加标记，直接通过最后 n 层，计算正常的交叉熵损失（掩码填充、编辑、保持正确）。
- **Branch 2（分配分支）**：从掩码位置中随机抽取一个捆绑 D（大小为 s～Uniform{1,...,16}），给 D 的位置加上标记向量，再次通过最后 n 层，仅对 D 计算掩码填充损失，其他位置仍从 Branch 1 的预测学习。训练时随机捆绑模拟了推理时的置信度选择束，使模型学会任何捆绑下的协调条件分布。

### 公式/算法流程（文字说明）
1. 给定去噪步骤的输入 (ctx, z)，计算前缀隐藏状态 h = Prefix(ctx, z)。
2. **Stage A**：h 经过最后一组 n 层 → ℓ_A = Head(LastN(h))，读取每个 masked 位置的最大概率，选择置信度 > τ_m2t 的集合 S（或最自信的单个位置）。
3. **Stage B**：若 S 非空，构造 h' = h + 1_S ⊙ m（m 是学习到的标记向量）；将 h' 再次通过 LastN 层 → ℓ_B = Head(LastN(h'))。用 ℓ_B 覆盖 ℓ_A 用于写入。
4. 使用 ℓ_B 执行 M2T（mask→token）写入和 T2T（token→token）编辑，写入相应位置。

推理时 K=1（一次协调），训练时支持 K 轮迭代（文中仅用 K=1）。

## 6. 论文的主要结论与发现

- CoCommit 在所有六个基准上均取得正向提升，其中**推理和精确答案任务提升最大**（DROP +4.38，CMATH +2.01，AIME +3.33），而知识型任务提升较小（TriviaQA +0.92，MMLU-Pro +0.15，GSM Plus +0.50）。
- 这表明**因子化误差在需要多 token 协调的任务中尤为严重**（如数学推理中的步骤一致性），而 CoCommit 通过联合模式解码减轻了该误差。
- 方法可以在不增加额外模型、不改变主干网络权重的情况下，仅通过轻量 LoRA 和标记向量实现有效协调。
- 作者强调**步内协调（within-step）与步间修正是互补的**，CoCommit 并不替代修正式后处理，而是从源头减少需要修复的不一致元组。

## 7. 优点

- **理论清晰**：从条件总相关性和模式平均化角度解释了并行解码的误差来源，为改进提供了坚实理论依据。
- **实现简洁**：重用骨干网络最后几层，仅需学习一个标记向量和 LoRA 适配器，无辅助模型，推理时增加一次部分前向计算（约 < 1/n 的总 FLOPs，n=3层，总20层）。

- **可扩展性**：支持迭代 K 次协调（文中仅实验 K=1），理论上可逼近联合扩散模型；也支持标记可见 token 进行联合修订。
