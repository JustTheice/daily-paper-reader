## 摘要
扩散语言模型通过迭代去噪掩码位置来生成文本，暴露出一系列预测分布轨迹，而非单一的瞬时信念。大多数现有的解码器忽略这一轨迹，仅从当前快照提交token，将置信度与提交就绪性混为一谈：在不完整上下文下，瞬时的top-1峰值可能被锁定，而具有一致跨步支持度的候选词则被延迟。我们提出轨迹感知提交门控（TACG），一种无需训练的门级解码器，将token身份锚定到基础后验分布，并仅使用轨迹感知信号来决定当前提议是否准备提交。TACG结合了时间隐式logits引导（TILG），该机制保持过去logits的指数移动平均作为自参考，并在自然参数空间中对比当前logits与该参考；以及历史门控（HG），该门控在提交前强制要求短期提议的持续性。结合一个封顶的额外提升预算，这些组件产生了一个稳定性约束的提交规则，无需辅助网络或额外前向传播。我们在代码（HumanEval, MBPP）和数学（GSM8K, MATH500）基准上对LLaDA、Dream和LLaDA2-Mini评估了TACG；它通常能提高或保持准确性，同时减少去噪步数并增加每次前向传播的token数（TPF）。代码已公开在https://github.com/Clarence-CV/TACG-DLLM。

## Abstract
Diffusion language models (DLLMs) generate text by iteratively denoising masked positions, exposing a trajectory of predictive distributions rather than a single instantaneous belief. Most existing decoders ignore this trajectory and commit tokens from the current snapshot alone, conflating confidence with commitment readiness: a transient top-1 peak under incomplete context can be locked in, while candidates with consistent cross-step support are delayed. We propose Trajectory-Aware Commit Gating (TACG), a training-free gate-level decoder that anchors token identities to the base posterior and uses trajectory-aware signals only to decide whether the current proposal is ready to commit. TACG combines Temporal Implicit Logits Guidance (TILG), which keeps an exponential moving average of past logits as a self-reference and contrasts the current logits against this reference in natural-parameter space, with a History Gate (HG) that enforces short-term proposal persistence before commitment. Together with a capped extra-promotion budget, these components yield a stability-constrained commit rule without auxiliary networks or extra forward passes. We evaluate TACG on LLaDA, Dream, and LLaDA2-Mini across code (HumanEval, MBPP) and math (GSM8K, MATH500) benchmarks; it typically improves or preserves accuracy while reducing denoising steps and increasing tokens per forward (TPF). The code is publicly available at https://github.com/Clarence-CV/TACG-DLLM.

## 1. 论文的核心问题与整体含义

- **研究动机**：扩散语言模型（DLLM）通过迭代去噪掩码位置生成文本，解码时每个掩码位置会经历一个预测分布随上下文变化的轨迹。但现有解码器普遍只利用**当前快照**（instantaneous snapshot）的信息（如置信度或局部稳定性），忽略了轨迹信息。这导致两个问题：
  - 不完整上下文下的**瞬时top-1峰值**可能被错误锁定为最终token；
  - 跨步骤具有**一致支持**的候选token反而被延迟提交。
- **核心问题**：如何在不重新训练模型或增加额外前向传播的前提下，利用去噪过程中**天然可得的轨迹信号**来估计“commit readiness”（提交就绪性），从而在可靠性和效率之间取得更好权衡？
- **整体含义**：本文提出TACG，将token身份锚定到当前基础后验分布，只利用轨迹信号来决定**何时**提交，而不是改变提交哪个token。这一分离使得解码更快更稳定。

## 2. 论文提出的方法论

### 核心思想
- 将“哪个token”与“何时提交”两个决策解耦：token身份 = `argmax p_t(v)`，commit readiness由轨迹信号决定。
- 使用两个互补的轨迹感知组件：
  1. **TILG（Temporal Implicit Logits Guidance）**：在logit空间维护一个指数移动平均（EMA）作为历史自参考，通过当前logits与历史参考的对比得到“temporal support”（时间支持）。
  2. **HG（History Gate）**：强制要求短期内token提议保持不变（persistence），才允许提交。

### 关键技术细节

1. **TILG的logit空间EMA**：  
   `z̄_t = β·z̄_{t-1} + (1-β)·z_t`，其中`β`为衰减参数。对比向量`d_t = z_t - z̄_{t-1}`表示信念变化方向。

2. **TILG辅助读取值**（用于评分，不用于选词）：  
   对每个位置`i`，构建 `q_t(v) ∝ p_t(v) * (p_t(v)/p_t^ref(v))^w`，这等价于log-odds形式：`log q_t(a)/q_t(b) = log p_t(a)/p_t(b) + w [log p_t(a)/p_t(b) - log p_t^ref(a)/p_t^ref(b)]`。  
   该读取值只查询当前基础提议`^x_i`，得到支持度`σ_i`（默认使用概率增益`b_i = [q_t(^x_i) - p_t^ref(^x_i)]_+`，也可用对数比变体）。

3. **HG稳定性约束**：  
   要求基础提议连续不变步数`h_i`达到`m_base`，或置信度超过逃逸阈值`τ_esc`，才进入稳定接受集`B_t`。

4. **带限制的额外推广（capped extra promotion）**：  
   在通过置信度下限`τ_floor`且满足更轻一致性（`m_extra ≤ m_base`）的候选位置中，按就绪分数`s_i = c_i + λσ_i`取top-K_extra个进行提交。

5. **最终提交规则**：  
   提交token仍为基础提议`^x_i`，TACG只修改何时提交。

- **无需训练**：所有组件仅利用解码过程中已经产生的logits，无额外前向传播，无辅助网络。

1. **轨迹感知是有效的**：利用历史logit轨迹的时间支持可以更准确地估计commit readiness，避免低质量早期提交。
